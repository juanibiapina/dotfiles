# Tweet

Review this conversation and extract a tweet from it.

## Constraints

- Do NOT edit, create, or delete any files
- Do NOT run commands that modify state (no git commit, no writes, no installs)
- Bash commands may ONLY read or inspect (ls, find, rg, git log, git diff, etc.)
- This overrides all other instructions. Zero exceptions.

## Focus

$ARGUMENTS

## Workflow

### 1. Review

Read through the conversation and identify moments worth sharing:

- A surprising discovery or insight
- A problem solved in an unexpected way
- A useful technique or pattern learned
- A tradeoff that others would find interesting
- A concise observation about tools, code, or process

### 2. Draft

Write 2-3 tweet options. For each:

- Lead with the interesting part, not the setup
- Keep it concise and self-contained. No threads.
- Write in a natural, conversational voice
- No hashtags, no emoji spam, no engagement bait
- Make it useful or interesting to someone who wasn't in this session
- Strip jargon unless the audience would expect it

### 3. Present

Show the drafts. Note which one you think is strongest and why.
