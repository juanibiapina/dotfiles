---
description: Tone down overly bold or visually aggressive designs. Reduces intensity while maintaining design quality and impact.
---

Reduce visual intensity in designs that are too bold, aggressive, or overstimulating, creating a more refined and approachable aesthetic. Use the frontend-design skill for design principles and anti-patterns.

$ARGUMENTS

## Understand Context First

Gather target audience, use cases, and brand personality from the conversation or codebase. If unclear, ask before proceeding. "Quieter" doesn't mean boring or generic — it means refined, sophisticated, and easier on the eyes.

## Assess

Identify intensity sources: overly saturated colors, extreme contrast juxtaposition, too many bold/heavy elements competing, excessive animation, too many visual elements/patterns, everything large and loud with no hierarchy.

## Refinement Dimensions

### Color
Reduce saturation to 70-85%. Soften palette to muted, sophisticated tones. Use fewer colors more thoughtfully. Let neutrals do more work, color as accent (10% rule). Use tinted grays instead of pure gray. Never gray on colored backgrounds.

### Visual Weight
Reduce font weights (900 → 600, 700 → 500). Decrease sizes where appropriate. Build hierarchy through weight, size, and space instead of color and boldness. Increase breathing room. Reduce border thickness and opacity, or remove borders entirely.

### Simplification
Remove decorative elements that don't serve purpose (gradients, shadows, patterns, textures). Reduce border radius extremes. Flatten visual hierarchy where possible. Reduce or remove blur effects, glows, and multiple shadows.

### Motion
Shorter distances (10-20px instead of 40px), gentler easing. Remove decorative animations, keep functional motion. Use ease-out-quart for smooth, understated movement. Remove animations entirely if they're not serving a clear purpose.

### Composition
Smaller contrast between sizes for a calmer feeling. Bring rogue elements back into systematic grid alignment. Replace extreme spacing variations with consistent rhythm.

## Constraints

- Don't make everything the same size/weight — hierarchy still matters
- Don't remove all color — quiet is not grayscale
- Don't eliminate personality — maintain character through refinement
- Don't sacrifice usability — functional elements still need clear affordances
- Some visual anchors are needed — not everything can be light and small
