Identify and improve unclear, confusing, or poorly written interface text to make the product easier to understand and use.

$ARGUMENTS

## Assess

Find clarity problems: jargon users won't understand, ambiguous text, passive voice, text that's too wordy or too terse, missing context, tone mismatches. Understand the audience, their mental state during each interaction, and any space constraints.

## Copy Areas to Review

### Error Messages
Explain what went wrong in plain language, suggest how to fix it, don't blame the user, include examples when helpful. "Email addresses need an @ symbol" beats "Invalid input."

### Form Labels & Instructions
Use clear specific labels (not generic placeholders). Show format expectations with examples. Explain why you're asking when not obvious.

### Buttons & CTAs
Describe the action specifically with verb + noun. "Create account" beats "Submit." "Save changes" beats "OK."

### Help Text & Tooltips
Add value beyond repeating the label. Answer the implicit "What is this?" or "Why do you need this?"

### Empty States
Explain why it's empty, show the next action clearly. "No projects yet. Create your first project to get started."

### Success Messages
Confirm what happened and explain what happens next. "Settings saved! Changes take effect immediately."

### Loading States
Set expectations about duration. "Analyzing your data... usually takes 30-60 seconds."

### Confirmation Dialogs
State the specific action and consequences. "Delete 'Project Alpha'? This can't be undone." Use clear button labels ("Delete project" not "Yes").

## Principles

1. **Be specific**: "Enter email" not "Enter value"
2. **Be concise**: Cut unnecessary words without sacrificing clarity
3. **Be active**: "Save changes" not "Changes will be saved"
4. **Be human**: "Oops, something went wrong" not "System error encountered"
5. **Be helpful**: Tell users what to do, not just what happened
6. **Be consistent**: Same terms throughout — don't vary for variety

## Constraints

- Don't use jargon without explanation
- Don't blame users ("You made an error" → "This field is required")
- Don't use placeholders as the only labels — they disappear when typing
- Don't repeat information already visible on screen
- Don't use humor for errors — be empathetic instead
