Create or improve onboarding experiences that help users understand, adopt, and succeed with the product quickly. Get users to their "aha moment" as fast as possible.

$ARGUMENTS

## Assess

Understand what users need to learn: What are they trying to accomplish? Where do they get stuck? What's the "aha moment"? What's the minimum they need to learn to succeed?

Consider the users: experience level, motivation, time commitment, familiarity with alternatives.

## Onboarding Principles

- **Show, don't tell**: Demonstrate with working examples, not descriptions. Use real functionality, not separate tutorial mode.
- **Make it optional**: Let experienced users skip. Don't block access to the product.
- **Time to value**: Front-load the most important 20%. Save advanced features for contextual discovery.
- **Context over ceremony**: Teach features when users need them, not all upfront. Empty states are onboarding opportunities.
- **Respect intelligence**: Be concise and clear, don't patronize or over-explain.

## Onboarding Types

### Initial Product Onboarding
Welcome screen with clear value proposition and time estimate. Minimal account setup (collect more later). Introduce 1-3 core concepts interactively. Guide to first real success with pre-populated examples or templates.

### Feature Discovery
Empty states that explain what will appear, why it's valuable, and how to create the first item. Contextual tooltips at the moment of relevance (dismissable, with "Don't show again"). Progressive disclosure — unlock complexity gradually.

### Guided Tours (for complex interfaces)
Spotlight specific UI elements (3-7 steps max). Interactive over passive (let users click real buttons). Focus on workflow ("Create a project") not features ("This is the project button"). Provide sample data and make tours replayable from help menu.

## Empty State Design

Every empty state needs: what will be here, why it matters, how to get started (CTA), and visual interest. Handle different empty types differently: first use (emphasize value), user cleared (light touch), no results (suggest different query), no permissions (explain why), error (explain + retry).

## Constraints

- Don't force long onboarding before users can access the product
- Don't show the same onboarding twice — track completion and respect dismissals
- Don't create separate tutorial modes disconnected from the real product
- Don't overwhelm with information upfront — progressive disclosure
- First 30 seconds determine whether users stay — invest disproportionately there
